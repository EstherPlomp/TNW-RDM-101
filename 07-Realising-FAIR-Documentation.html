8  Realising FAIR: Data Documentation

TNW RDM 101 course

In this module, we will focus on the realisation of FAIR data. We will explore some best practices and tools that can facilitate the implementation of the FAIR principles. At the end of this module you should be able to: * Recognize the relevance of documentation and be familiar with diverse tools to document data * Identify relevant tools to advance towards FAIR data

8.1 Activities

  • Watch the videos about ‘Data Documentation tools’ and complete the corresponding quizzes.
  • Write a readme file and share it on GitHub. Comment on another person’s readme and provide them with some tips to improve.

8.2 Documentation tools

Now that you know more about how to organise your data and name your files, we can talk about documentation. Documentation is vital to making your work understandable, which is, in turn, necessary for your work to be reusable. Documentation is important not only for data, but for your projects in general, including the code you write. Your documentation should provide context for your project and its data and should, for example, provide information about the data collection, structure, and ownership.

In the next video we will talk about different methods and tools you can use to document projects, datasets, and code. We will also briefly talk about metadata. 04-2_Module-4_Video_Presentation_DataDocumentation]

## Assignment: Create a ReadMe file Estimated time: 15 minutes

  1. Download the Cornell template for a ReadMe file.
  2. Adapt and start filling in the template to describe a dataset of your project (you are allowed to delete and add new fields)
  3. If you are unsure about the meaning of a component of the ReadMe file, go to: Guide to writing “ReadMe” style metadata - Cornell University
  4. When you’re finished, share your template on GitHub.
  5. Pick a ReadMe file of someone else and read it. Provide feedback on the clarity of the ReadMe.

8.3 Assignment: Data Dictionaries (for tabular data - optional)

Estimated time: 15 minutes 1. Watch a video on data dictionaries 2. Go to a tabular dataset you are working on or have worked on. 3. In an excel spreadsheet, start creating your data dictionary. Ave the data dictionary in the same folder as the dataset you’re describing. List all the variables that you find in the dataset on column A. Then, add a clear and understandable description of that variable on column B.

Note that you can also create a package for your data with Frictionless Data

8.5.1 Quiz Data Documentation tools

Test your knowledge after watching the ‘Data Documentation tools’ video. Indicate if the following sentences are True or False. 1. Metadata - data about data - is an important type of documentation (Answer: True) 2. README files can be used only to document whole projects (Answer: False) 3. Codebooks are used for the documentation of code (Answer: - False) 4. GitHub/GitLab and Jupyter notebooks are good tools for documenting code (Answer: True) 5. Metadata can be added to data as part of the file name, using a README file and using tags (Answer: True)
